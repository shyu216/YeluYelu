# YeluYelu 重构计划

**日期**: 2026-04-08

## 1. 项目改造成 Vue SPA 并部署到 GitHub Pages

### 目标
将现有项目改造为 Vue 3 单页应用 (SPA)，使用纯静态后端部署到 GitHub Pages。

### 实施方案
- 使用 Vite + Vue 3 构建项目
- 路由使用 Vue Router
- 状态管理使用 Pinia
- 部署配置 GitHub Actions Workflow：
  - 触发条件：push 到 main 分支
  - 构建步骤：npm install → npm run build
  - 部署步骤：使用 gh-pages 或直接部署到 GitHub Pages

## 2. GitHub Auth 集成

### 目标
支持用户通过 GitHub OAuth 进行注册/登录，然后支持提交 Pull Request。

### 实施方案
- 使用 GitHub OAuth App 进行身份认证
- 登录后获取用户 GitHub 账号信息
- 基于 GitHub Token 实现 PR 创建功能
- 可选：使用第三方库如 `vue-auth` 或手動实现 OAuth 流程

### 配置步骤

#### 1. 创建 GitHub OAuth App
1. 进入 GitHub Settings → Developer settings → OAuth Apps
2. 点击 "New OAuth App"
3. 填写信息：
   - Application name: 夜鹭页录
   - Homepage URL: https://your-username.github.io/yeluyelu
   - Authorization callback URL: https://your-username.github.io/yeluyelu/callback
4. 获取 Client ID

#### 2. 配置前端
修改 `src/stores/auth.js` 中的 `GITHUB_CLIENT_ID` 为你的 Client ID

#### 3. (可选) 获取 Client Secret
如果需要更安全的 token 交换，需要后端服务处理

## 3. 封装图片上传为 Pull Request

### 目标
用户登录后可以直接上传图片，系统自动创建 PR。

### 实施方案
- 前端：图片上传组件，支持拖拽上传
- 图片处理功能：
  - Resize 调整尺寸
  - 格式转换 (支持 JPG、PNG、WebP 等)
  - 使用 Canvas 或库如 `browser-image-compression` 进行处理
- 后端逻辑（静态部署时通过 GitHub API）：
  - 使用用户 GitHub Token 创建新分支
  - 将处理后的图片提交到仓库
  - 自动创建 Pull Request
- 特定格式规范：
  - 目录结构：`uploads/{username}/{timestamp}_{original_name}`
  - 元数据记录：上传用户、时间、图片信息等

## 4. PR 自动合并机器人

### 目标
机器人监听 PR 事件，当 PR 符合特定条件时自动合并。

### 实施方案
- 使用 GitHub Actions 或独立机器人（如 Probot 插件）
- 触发条件：
  - PR 标题/描述包含特定标记
  - 仅包含图片文件
  - 文件路径符合预期格式
- 验证逻辑：
  - 检查 PR 是否只包含图片
  - 验证路径格式正确 (`uploads/{username}/*`)
  - 可选：图片内容安全检查
- 自动操作：
  - 检查通过后自动 `approve` + `merge`
  - 或直接 `squash and merge`

### 技术选型
- **方案 A**: 使用 GitHub Actions + 自定义 Action
- **方案 B**: 使用 Probot 创建自定义机器人
- **方案 C**: 使用 GitHub App + Webhook
