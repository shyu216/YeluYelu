## 3. 封装图片上传为 Issue（替代原 PR 方案）

### 目标
用户登录后可以直接上传图片，系统自动创建 Issue。

### 方案调整原因
- **原 PR 方案问题**：普通用户（未 Fork、未添加为协作者）无法直接提交 PR
- **Issue 方案优势**：任何登录用户都可以创建 Issue，无需额外权限

### 实施方案
- 前端：图片上传组件，支持拖拽上传
- 图片处理功能（可选）：
  - Resize 调整尺寸
  - 格式转换 (支持 JPG、PNG、WebP 等)
  - 使用 Canvas 或库如 `browser-image-compression` 进行处理
- 后端逻辑（通过 GitHub API）：
  - 使用用户 GitHub Token 创建 Issue
  - 图片作为 Issue 附件上传（通过 uploads API）
- Issue 格式规范：
  - 标题：`📸 上传: {名称} by {摄影师}`
  - 正文包含：拟态名称、摄影师、上传时间

### 技术细节
```javascript
// 创建 Issue
POST /repos/{owner}/{repo}/issues
{
  "title": "📸 上传: 哥斯拉 by 张三",
  "body": "## 夜鹭拟态图鉴上传\n\n### 拟态名称\n哥斯拉\n..."
}

// 上传图片附件
POST /repos/{owner}/{repo}/issues/{issue_number}/attachments
Content-Type: multipart/form-data
```

## 4. Issue 自动处理机器人

### 目标
机器人监听 Issue 事件，自动处理包含图片的 Issue。

### 实施方案
- 使用 GitHub Actions
- 触发条件：Issue 添加 `needs-review` 标签
- 处理逻辑：
  1. 提取 Issue 标题中的图片名称
  2. 下载 Issue 附件中的图片
  3. 保存到 `public/images/` 目录
  4. 更新 `data.json`
  5. 自动创建 PR → 合并到主分支
  6. 关闭 Issue 并通知用户

### 工作流文件
- `.github/workflows/auto-process-issue.yml`

### 验证规则
- Issue 标题格式：`提交 {名称}` 或 `📸 上传: {名称}`
- 必须包含图片附件
- 支持格式：jpg、png、gif、webp

## 5. 安全与审核

### 当前方案说明
- **Token 存储**：使用 Cookie 存储 GitHub Token
- **安全等级**：⚠️ 中等（纯前端方案，无法完全保护 Token）
- **权限范围**：需要 `public_repo` 权限来创建 Issue 和上传附件

### 可选的增强方案
1. **Webhook 审核**：Issue 创建后添加审核标签，由管理员确认后才处理
2. **自动审核**：只接受特定格式的 Issue，自动拒绝不合规的
3. **内容过滤**：使用 AI 或规则过滤不当内容
